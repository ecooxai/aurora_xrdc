# Overview

## Current status

`vibe_rdesk` is now a Rust-based remote desktop MVP. It starts an HTTP server, serves a browser client, captures the current X11 desktop through FFmpeg, prefers NVIDIA NVENC for H.264/H.265 when available, falls back to CPU encoders, and streams encoded frames to the browser over WebSocket for WebCodecs decode.

The project currently supports:

- H.264 default streaming path.
- H.265 best-effort streaming path.
- VP8 streaming path.
- Browser-side touchpad and click controls relayed through `xdotool`.
- Client-selectable codec, bitrate, and FPS.
- `?debug=1` overlay with latency, FPS, codec, encoder mode, CPU usage, memory usage, and network speed.
- Service worker caching for the web client.
- Bottom error notification UI without `alert()` or `prompt()`.

## Notes

- The capture pipeline targets X11 via `ffmpeg -f x11grab`.
- GPU preference currently means NVENC when FFmpeg exposes `h264_nvenc` or `hevc_nvenc`; other systems fall back to CPU encode.
- H.265 browser support depends heavily on the client browser and OS codec stack.
- The current server spawns one FFmpeg capture pipeline per connected browser session.

## Project structure

```text
.
├── .gitignore                  # Ignore Rust build outputs and temp artifacts.
├── Cargo.lock                  # Dependency lockfile generated by Cargo.
├── Cargo.toml                  # Rust package manifest and dependencies.
├── dev.sh                      # Run tests first, then start the app.
├── test.sh                     # Run cargo test and cargo build.
├── doc
│   ├── common_mistakes.md      # Guardrails for agent work.
│   ├── overview.md             # Project status, structure, and file tree.
│   ├── update.md               # Task history with timestamps.
│   └── vibetask.md             # Human-maintained task input file.
├── dist                        # Reserved output folder.
├── src
│   ├── annexb.rs               # H.264/H.265 Annex B and VP8 IVF frame parsing helpers.
│   ├── app.rs                  # HTTP routes, asset serving, and WebSocket entrypoint.
│   ├── ffmpeg.rs               # FFmpeg encoder selection, process spawning, and xdotool helpers.
│   ├── main.rs                 # Binary entrypoint and logging setup.
│   ├── messages.rs             # WebSocket message schema.
│   ├── session.rs              # Per-client streaming and input control session flow.
│   ├── settings.rs             # Shared server and stream configuration types.
│   ├── streamer.rs             # Capture process output reading and encoded frame forwarding.
│   └── system_stats.rs         # CPU, memory, and network telemetry sampling.
└── web
    ├── app.css                 # Mobile-friendly client styling.
    ├── app.js                  # WebCodecs client, controls, debug overlay, and notifications.
    ├── index.html              # Browser UI shell.
    └── sw.js                   # Service worker cache/update logic.
```
